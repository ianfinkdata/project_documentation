# Claude Code Instructions

## Project conventions

Active production projects use `p-` prefixed ticket numbers (e.g., `p-20001`).

`J-` ticket projects and projects in `projects/paused/` and `projects/complete/` are legacy demo projects used to establish and validate the folder structure. Do not reference, modify, or create files in these projects unless the user explicitly mentions a J-ticket or demo project by name.

All net new work goes under `projects/active/` with a `p-` ticket number.

## Power BI project conventions

Power BI `.pbip` project folders are named `pbip_[filename]` where `[filename]` is the `.pbip` filename without the extension, in snake_case.

Example: a file named `Sales Pipeline Demo.pbip` lives in a folder named `pbip_sales_pipeline_demo/`.

This keeps the report identity visible in directory listings and supports multiple `.pbip` projects within the same ticket folder.

The `.pbi/` folder generated by Power BI Desktop (`cache.abf`, `localSettings.json`, `editorSettings.json`) contains temp files and is gitignored — do not commit these files.
